\item \doxysection*{How to compile}
\end{DoxyItemize}

First, clone the repo to your local machine.

\doxysubsection*{Using {\ttfamily Makefile}\+:}


\begin{DoxyItemize}
\item open a terminal
\item move to the project folder
\item type {\ttfamily make} to build the executable {\ttfamily exe}
\item {\ttfamily ./exe.x} from command line
\end{DoxyItemize}

\doxysubsection*{From command line\+:}


\begin{DoxyItemize}
\item {\ttfamily \$ g++ -\/o exe.\+x -\/Wall -\/Wextra -\/g -\/std=c++14 -\/I include main.\+cpp}
\item then do {\ttfamily ./exe.x}
\end{DoxyItemize}

\doxysubsection*{To check memory leaks with \href{https://valgrind.org}{\texttt{ Valgrind}} and see the heap/leak summary\+:}


\begin{DoxyItemize}
\item {\ttfamily valgrind ./exe.x -\/-\/leaks-\/check=full}
\end{DoxyItemize}

The current version doesn\textquotesingle{}t leak memory, as can be verified.

\doxysection*{Unit testing}

To check the goodness of our functions, we used the testing framework \href{https://github.com/google/googletest}{\texttt{ Google Test}}. In particular, we provide tests for the each one of the implemented classes. If you have Google Test installed on your machine, and for instance you want to compile and run the tests for the \mbox{\hyperlink{structNode}{Node}} class, just type from command line\+:

\item {\ttfamily g++ Node\+Testing.\+cpp -\/lgtest -\/lgtest\+\_\+main -\/pthread -\/o Node\+Testing -\/std=c++14}
